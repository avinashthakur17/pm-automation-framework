Human Inputs Required to Start
Assumptions made — one-time setup inputs before the agent mesh runs autonomously

Business objectives

  • Primary KPI: target CPA or target ROAS per channel
  • Revenue target (monthly/quarterly)
  • Acceptable CPA ceiling (hard stop rule)
  • Growth vs. efficiency mode toggle

Budget plan

  • Total monthly budget by channel (Google / Meta / Affiliates)
  • Daily budget per campaign (floor and ceiling)
  • Maximum daily reallocation limit (%)
  • Seasonal uplift calendar (peak dates and uplift %)

Campaign configuration

  • Campaign objective per channel (conversions, leads, traffic)
  • Campaign type (PMax, Search, Advantage+, etc.)
  • Conversion window (days) per channel
  • Attribution model selection

Audience & targeting

  • ICP definition (demographics, interests, intent signals)
  • High-LTV seed list (CSV from CRM — min. 1,000 records)
  • Geo inclusions and exclusions
  • Frequency cap ceiling per channel

Creative assets

  • Brand guidelines (colors, fonts, tone, logo files)
  • Offer brief (discount, CTA, key value props)
  • Initial creative pool (min. 3 images + 2 videos)
  • Prohibited content list (legal, competitor mentions)

Measurement & tracking

  • Conversion event names (purchase, lead, add-to-cart)
  • Dynamic revenue value: yes/no + implementation method
  • Appsflyer app ID and event mapping (if mobile)
  • GA4 property ID and goal event names

Platform access

  • Google Ads account IDs + API OAuth credentials
  • Meta Ad Account IDs + System User token
  • Appsflyer API token + app IDs
  • DataBricks workspace URL + service principal

Alert & escalation config

  • KPI alert thresholds (e.g. CPA >50% above target = critical)
  • Escalation routing (who gets WA vs. email vs. Slack)
  • Agent autonomy guardrails (bid cap ±%, budget shift ±%)
  • Review cadence preference (daily queue review time)

CI
Creative Intelligence Agent
Scoring · rotation · fatigue detection · brief generation · dispatches to Creative Generation Sub-Agent
▼
Sub-agentsCreative Generation Sub-Agent (video + audio + image) · Image Asset Manager · Copy Optimizer — all dispatched via Orchestrator
Pre-launch scoringAny new brief (human-submitted or AI-generated on fatigue) → Pencil AI API scores predicted CTR, CVR, fatigue timeline → score + brief surfaced to human in Slack approval queue. Below-threshold concepts blocked before generation spend is committed.
Generation dispatchOn human brief approval: publishes task to Orchestrator with brief payload, format specs, channel targets, and brand guardrail config. Orchestrator dispatches to Creative Generation Sub-Agent. Agent returns draft assets to approval queue — nothing goes live without human sign-off.
Rotation logicReads gold.creative_performance every 2 hrs. Thompson Sampling (multi-armed bandit) distributes impressions. At 95% statistical confidence → promotes winner, deprioritises others via platform API. No single variant exceeds 70% impression share before confidence is reached.
Fatigue detectionImage: 7-day CTR drops >25% vs. peak → auto-pause + generation brief triggered. Video: 7-day CVR drops >30% vs. peak → GPT-4o classifies root cause (hook failure / offer fatigue / audience saturation) → tailored brief sent to Generation Sub-Agent after human approval.
Image post-processingAuto-crops generated or uploaded master images to all placement ratios (1:1, 4:5, 9:16, 1.91:1) via Bannerbear API. Validates Meta text-to-image ratio. Uploads via platform asset APIs.

CG
Creative Generation Sub-Agent
AI video generation · AI audio & voiceover · image generation · caption production · human approval gate before any asset goes live
▼
TriggerDispatched by Creative Intelligence Agent on: (1) human-approved new brief, or (2) fatigue alert brief approved by human. Never self-triggers.
InputsStructured brief from Redis: {offer, hook_angle, audience_segment, visual_direction, format_list, duration_seconds, brand_guidelines_url, prohibited_content, channel_targets, winning_element_history}. Brand guidelines and asset library fetched from DataBricks gold.brand_assets.
Video generation Step 1 — Script: GPT-4o writes hook script (0–3s), narrative (3–25s), and CTA (final 3s) based on brief. Hook variants: 3 options generated for human selection.
Step 2 — Voiceover: ElevenLabs API generates voiceover from approved script. Voice clone matched to brand voice profile stored in ElevenLabs project. Emotion and pacing configured per audience segment (urgency for retargeting, informational for prospecting).
Step 3 — Visual generation: Runway ML Gen-4 generates video from text prompt + reference image (brand product shot). Kling AI used for image-to-video animation of static product images. HeyGen generates AI spokesperson variant if brief calls for UGC-style.
Step 4 — Music / SFX: Suno AI generates background music matched to ad mood (energetic / calm / urgent). ElevenLabs SFX for any sound effects. Mixed at -14 LUFS for platform compliance.
Step 5 — Assembly: FFmpeg combines voiceover + video + music. AssemblyAI generates SRT captions from final audio. Validates sync. Exports: 9:16 (Reels/TikTok), 4:5 (Feed), 1:1 (Square), 16:9 (YouTube).
Audio ad generation For audio-first channels (Spotify, YouTube audio, podcast pre-roll):
Script: GPT-4o writes 15s / 30s audio ad script with clear sonic branding hook and verbal CTA.
Voiceover: ElevenLabs generates primary read + 2 tone variants (enthusiastic / conversational / authoritative).
Music bed: Suno AI generates branded background track matching brand guidelines tempo. Ducked under voiceover at -18 dB.
Final mix: Exported as MP3 (128kbps) and WAV (48kHz/16-bit) for platform compatibility.
Image generationGPT-4o generates detailed image prompt from brief. Midjourney API (or DALL-E 3 as fallback) generates 4 variants. Branded elements (logo, product) composited via Cloudinary API using template overlays. Text overlays applied via Bannerbear. All variants auto-cropped to required placement ratios.
Brand guardrailsBefore generation: brief validated against prohibited content list from Redis config. After generation: GPT-4o Vision reviews each asset for brand compliance (logo placement, color palette adherence, prohibited imagery). Non-compliant assets flagged and regenerated — never surfaced to human approval queue in non-compliant state.
Human approval gateAll generated assets (video, audio, image) delivered to Slack approval channel as: preview link + generation parameters + Pencil AI predicted score + brand compliance status. Human actions: APPROVE (→ Creative Intel. Agent handles upload and rotation) / REJECT (→ brief returned with rejection reason for regeneration) / REVISE (→ human edits brief fields, sub-agent regenerates specific elements only). Nothing reaches platform without explicit APPROVE.
StorageAll generated assets stored in S3 (or GCS) with metadata: brief_id, generation_model, generation_params, approval_status, platform_upload_ids, performance_outcome (updated post-launch). Enables model selection optimization over time based on which generation tool produces highest-performing assets.

AI creative generation & testing (video + audio + image) Open full diagram ↗ ▼
H
Human — brief submission
Submits creative brief via Slack form or config UI: offer, audience pain point, visual direction, format types needed (video / audio / image), channel targets, duration, and any reference assets. No raw media required — brief is the only input.
1
Creative Intelligence Agent — pre-scoring
Pencil AI scores brief concept against account historical win/loss data → returns predicted CTR, CVR, and fatigue timeline. Brief enriched with winning element history from DataBricks (top hook angles, best CTA verbs, highest-CVR offer framing). Score + enriched brief surfaced in Slack for human review.
H
Human — brief approval (gate 1)
Reviews Pencil AI score and enriched brief. Actions: APPROVE (proceed to generation) / REVISE (edits brief fields inline in Slack) / REJECT (brief closed). If score is below threshold, agent flags this prominently — human must explicitly override to proceed.
2
Creative Generation Sub-Agent — script & prompt writing
GPT-4o writes: (a) Video script — 3 hook variants (0–3s each), narrative (3–25s), CTA (final 3s). (b) Audio ad script — 15s and 30s versions with verbal hook and CTA. (c) Image generation prompt — detailed visual description for each format variant. All checked against prohibited content list before proceeding.
3
Creative Generation Sub-Agent — parallel generation
Runs in parallel across asset types:
Video: ElevenLabs generates voiceover from approved script → Runway ML Gen-4 generates video footage → Kling AI animates product image for hook → HeyGen generates spokesperson variant if brief calls for UGC style → FFmpeg assembles all layers (video + VO + music) → exported in all required ratios (9:16 / 4:5 / 1:1 / 16:9).
Audio: ElevenLabs generates 2–3 voice tone variants → Suno AI generates brand-matched music bed → FFmpeg mixes at -14 LUFS → exported as MP3 + WAV.
Image: Midjourney API generates 4 variants → Cloudinary composites brand elements → Bannerbear applies text overlays and crops to all placement ratios.
AssemblyAI auto-generates SRT captions for all video and audio assets.
4
Creative Generation Sub-Agent — brand compliance check
GPT-4o Vision reviews each generated asset: logo placement, color palette adherence, prohibited imagery, Meta text density, platform spec compliance (aspect ratio, file size, duration limits). Non-compliant assets auto-regenerated with corrected parameters. Only compliant assets advance to approval queue.
H
Human — asset approval (gate 2)
Slack approval message contains: preview link for each asset, generation model used, Pencil AI predicted score, brand compliance status, and suggested channel targets. Human actions: APPROVE ALL / APPROVE SELECTED (numbered) / REJECT (with reason) / REVISE (specifies which element to regenerate — e.g. "regenerate hook only" or "change VO tone to calmer"). Rejected assets loop back to step 3 with revision notes.
5
Creative Intelligence Agent — launch prep & upload
Uploads approved assets to platform asset libraries via Google Asset API and Meta Creative Hub API. Assigns to active ad sets. Adds assets to rotation pool. Logs asset metadata (brief_id, generation_model, generation_params, approval_timestamp) to DataBricks gold.creative_assets.
6
Creative Intelligence Agent — rotation & optimization
Every 2 hrs: Thompson Sampling distributes impressions across all variants (AI-generated and human-produced equally — no bias). At 95% statistical confidence → winner promoted, others deprioritised. Performance outcomes written back to DataBricks to train future brief generation and Pencil AI scoring.
7

7-day CTR/CVR trend monitored continuously. On fatigue trigger: GPT-4o classifies root cause (hook failure / offer fatigue / audience saturation) → generates targeted brief incorporating winning element history → dispatches to Slack with recommended generation parameters pre-filled. Human approves brief → loop restarts at step 1.

Protocols, Guardrails & Safety Rules
Standards governing agent communication, autonomy limits, and data safety

A2A — Agent-to-Agent Protocol

  • All inter-agent messages use Google A2A JSON envelope
  • Fields: agent_id, task_type, payload, confidence_score, rollback_fn, timestamp
  • Transport: Redis Pub/Sub (topic per domain: campaign / creative / audience / report)
  • Orchestrator is the only agent that subscribes to all topics
  • Subagents publish results; Orchestrator decides action or escalation

MCP — Model Context Protocol

  • Each platform integration exposed as an MCP server (Google Ads, Meta, Appsflyer)
  • Agents call tools via MCP — standardized input/output schemas
  • Enables tool discovery, permission scoping, and rate limit management
  • All MCP tool calls logged before execution for audit trail
  • Failed calls → retry with exponential backoff → escalate to human

Data security & access

  • All platform API credentials stored in HashiCorp Vault (or AWS Secrets Manager)
  • Agents access credentials at runtime via secret reference — never hardcoded
  • DataBricks service principal scoped to read-only for most agents; write access only for Data Sync
  • Platform API write access scoped per agent (Campaign Opt. cannot call Creative API)
  • All PII (CRM data) processed within DataBricks — never sent to external LLM APIs

Human-in-the-loop triggers

  • Confidence score <0.75 on any recommended action
  • Budget shift >15% of any single campaign's daily budget
  • Campaign pause or new campaign creation
  • Bid strategy type change (e.g. tCPA → tROAS)
  • Creative concept approval (pre-production)
  • Attribution model or conversion event change
  • Any action during campaign learning period
  • Creative brief approval before AI generation begins (gate 1)
  • Generated asset approval before platform upload (gate 2)
Autonomy guardrails — hard limits encoded in Orchestrator
Bid adjustmentsMaximum ±20% change per campaign per 2-hr cycle. Cumulative daily cap: ±40% from start-of-day bid.
Budget reallocationMaximum ±15% shift of any campaign's daily budget per cycle. Minimum floor budget enforced per campaign (set by human).
Auto-rollback triggerCPA increases >20% vs. pre-action baseline in the following 2-hr cycle → automatic rollback via platform API.
Creative rotationNo single creative variant can receive >70% impression share until 95% statistical confidence reached. Prevents premature concentration.
Negative keywordsAuto-add only when query appears >3 times with 0 conversions and >$X spend (configurable). Brand and competitor terms always queued for human review.
Learning period lockZero automated structural changes during platform learning period. Agent monitors status flag; re-queues all pending changes for post-learning execution.
AI generation gateZero AI-generated assets (video, audio, or image) reach any platform without explicit human APPROVE in Slack. Generation runs autonomously but publication is always human-gated. Rejected assets cannot be re-submitted without brief revision.
Brand compliance blockAssets failing GPT-4o Vision brand compliance check are auto-regenerated with corrected parameters. Non-compliant assets are never surfaced to the human approval queue — they are silently regenerated first.
